# /plan - 功能实现规划

用于规划新功能的实现步骤。

## 使用方法

```
/plan "添加每日签到功能"
```

## 执行步骤

1. **需求分析**
   - 功能目标
   - 输入输出
   - 边界条件

2. **技术设计**
   - 需要修改的文件
   - 新增的文件
   - 数据库变更

3. **实现步骤**
   - 分步任务列表
   - 每步的预期结果

4. **测试计划**
   - 单元测试
   - 集成测试
   - 手动测试

## 输出格式

```markdown
## 功能：[功能名称]

### 涉及文件
- 修改：src/game/xxx.js
- 新增：src/game/newfeature.js
- 测试：test-newfeature.js

### 实现步骤
1. [ ] 步骤一
2. [ ] 步骤二
...

### 测试用例
- [ ] 正常情况
- [ ] 边界情况
- [ ] 错误情况
```
